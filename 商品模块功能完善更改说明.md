# 商品模块功能完善更改说明

本次更改完善了商品模块的三个核心功能：资产所属权校验、事务控制、资产关联接口，提升了系统的安全性、数据一致性和功能完整性。

## 一、资产所属权校验（安全加固）

### 更改内容
1. **`create_product`方法**：
   - 在关联资产时，新增所属权校验，确保用户只能关联自己拥有的资产
   - 校验逻辑：遍历所有asset_ids，检查`asset.user_id == current_user_id`
   - 异常：如果资产不属于当前用户，返回403错误`无权限操作资产ID {asset_id}`

2. **`parse_product`方法**：
   - 当指定`asset_id`参数时，新增所属权校验
   - 确保用户只能解析自己资产的内容

3. **新增资产关联方法**：
   - 所有新增的资产关联操作都包含所属权校验
   - 包括`add_product_assets`、`remove_product_asset`、`update_asset_role`等方法

### 影响范围
- 所有涉及资产关联的操作都会进行所属权校验
- 不会影响正常使用，只有当用户尝试关联不属于自己的资产时才会报错

## 二、事务控制（数据一致性保障）

### 更改内容
1. **`create_product`方法**：
   - 使用`db.begin_nested()`开启嵌套事务
   - 商品创建和资产关联操作在同一个事务中
   - 如果关联资产失败，整个事务回滚，不会创建孤立的商品记录

2. **`delete_product`方法**：
   - 开启事务，先删除所有资产关联，再删除商品
   - 确保删除操作的原子性，不会出现商品已删除但关联还存在的情况

3. **`update_product`方法**：
   - 开启事务，确保分类更新等操作的原子性
   - 为后续可能的关联更新预留事务支持

4. **`upload_and_create_product`方法**：
   - 开启事务，确保资产创建和商品创建的原子性
   - 如果商品创建失败，事务回滚，已经创建的资产记录会被删除
   - 注意：已经上传到对象存储的文件需要后续异步清理

### 事务实现方式
```python
try:
    with db.begin_nested():
        # 执行多个数据库操作
        operation1()
        operation2()
    # 事务自动提交
except Exception as e:
    db.rollback()  # 事务自动回滚
    raise e
```

### 影响范围
- 所有涉及多个数据库操作的方法都具备了原子性
- 不会影响现有业务逻辑，只会提升数据一致性

## 三、新增资产关联管理接口（功能扩展）

### 新增接口列表

#### 1. 批量添加关联资产
**接口**：`POST /products/{product_id}/assets`

**请求体**：
```json
{
  "asset_ids": [1, 2, 3],
  "asset_roles": {
    "1": "main",
    "2": "image",
    "3": "video"
  }
}
```

**功能**：为商品批量添加关联资产，支持设置资产角色
**权限**：仅商品所有者可操作
**校验**：所有资产必须属于当前用户

#### 2. 删除关联资产
**接口**：`DELETE /products/{product_id}/assets/{asset_id}?role=image`

**参数**：
- `role`（可选）：指定要删除的角色，不传则删除该资产的所有关联

**功能**：删除商品与资产的关联关系，不会删除资产本身
**权限**：仅商品所有者可操作

#### 3. 修改资产角色
**接口**：`PUT /products/{product_id}/assets/{asset_id}/role`

**请求体**：
```json
{
  "role": "main"
}
```

**功能**：修改商品关联资产的角色，如将普通图片设置为主图
**权限**：仅商品所有者可操作
**角色范围**：main/image/video/audio

#### 4. 获取商品关联资产列表
**接口**：`GET /products/{product_id}/assets?role=image`

**参数**：
- `role`（可选）：按角色筛选资产

**功能**：获取商品关联的所有资产列表
**权限**：仅商品所有者可操作
**返回**：资产详细信息列表

### 新增Service层方法
1. `add_product_assets` - 批量添加关联资产
2. `remove_product_asset` - 删除关联资产
3. `update_asset_role` - 修改资产角色
4. `list_product_assets` - 获取商品关联的资产列表

## 四、升级说明

### 兼容性
- 所有现有接口完全兼容，不会影响现有业务
- 新增接口是全新功能，不影响原有逻辑

### 数据库变更
- 无需修改数据库表结构，所有功能基于现有表实现
- 利用已有的`product_assets`表的`role`字段

### 建议测试点
1. 尝试关联不属于自己的资产，应该返回403错误
2. 创建商品时故意传入错误的asset_id，验证商品不会被创建
3. 测试新增的四个资产关联接口的功能正确性
4. 测试删除商品时，关联的资产关联也被删除
5. 测试修改资产角色功能，确保角色正确更新

2. 分类移动或重命名时，添加异步更新商品冗余字段的逻辑
3. 为资产关联操作添加批量删除接口
4. 支持一次修改多个资产的角色
