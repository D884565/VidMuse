# 多通道检索模块使用指南

## 概述
多通道检索模块是一个通用的检索框架，支持多种检索渠道，提供统一的检索接口，内置查询增强和结果后处理能力。

## 快速开始

### 1. 基本使用
```python
from backend.v1.app.search import SearchEngine, SearchQuery

# 初始化检索引擎（使用默认配置）
search_engine = SearchEngine()

# 构建查询
query = SearchQuery(
    query_text="如何重置设备密码？",
    top_k=5,
    filters={"product_id": "123"},
    metadata={"user_id": 456, "project_id": 789}
)

# 异步检索（推荐）
results = await search_engine.asearch(query)

# 处理结果
for result in results:
    print(f"[{result.source}] {result.content} (得分: {result.score})")
    print(f"元数据: {result.metadata}")
    print("-" * 50)
```

### 2. 自定义配置
```python
from backend.v1.app.search import SearchConfig, SearchEngine

# 自定义配置
custom_config = {
    "ENABLED_CHANNELS": ["vector_db", "mysql"],  # 只启用向量数据库和MySQL
    "DEFAULT_TOP_K": 20,
    "CHANNEL_CONFIG": {
        "vector_db": {
            "collection": "custom_collection",  # 使用自定义集合
            "weight": 1.2
        },
        "mysql": {
            "table": "custom_table",
            "search_fields": ["title", "content"]
        }
    },
    "ENABLED_POST_PROCESSORS": ["deduplicator", "reranker"],  # 只启用去重和重排序
    "POST_PROCESSOR_CONFIG": {
        "result_filter": {
            "min_score": 0.5  # 提高最低得分阈值
        }
    }
}

# 使用自定义配置初始化
config = SearchConfig.from_dict(custom_config)
search_engine = SearchEngine(config)
```

### 3. 动态添加组件
```python
from backend.v1.app.search import SearchEngine
from my_custom_channel import MyCustomChannel
from my_custom_processor import MyCustomProcessor

# 初始化引擎
search_engine = SearchEngine()

# 添加自定义检索渠道
custom_channel = MyCustomChannel(config={"api_key": "xxx"})
search_engine.add_channel(custom_channel)

# 添加自定义查询处理器
custom_processor = MyCustomProcessor()
search_engine.add_query_processor(custom_processor)
```

### 4. 健康检查
```python
# 检查所有渠道的健康状态
health_status = search_engine.health_check()
print(f"整体状态: {health_status['status']}")
print(f"健康渠道数: {health_status['healthy_channels']}/{health_status['total_channels']}")
for channel, status in health_status['channels'].items():
    print(f"  {channel}: {status}")
```

## 内置组件说明

### 检索渠道
1. **vector_db**: 向量数据库检索（支持ChromaDB、Milvus、Qdrant）
2. **mysql**: MySQL关系型数据库全文检索
3. **http_api**: 外部HTTP API检索

### 查询增强处理器
1. **context_processor**: 会话上下文补全
2. **query_rewriter**: 查询文本优化重写
3. **intent_recognizer**: 用户查询意图识别

### 结果后处理器
1. **result_filter**: 结果过滤（得分、来源、元数据）
2. **deduplicator**: 结果去重
3. **result_merger**: 相似结果合并
4. **reranker**: 结果重排序

## 扩展开发

### 自定义检索渠道
继承`SearchChannel`接口，实现以下方法：
- `channel_name`: 渠道唯一名称
- `channel_type`: 渠道类型
- `asearch`: 异步检索方法
- `health_check`: 健康检查方法

### 自定义处理器
继承`BaseQueryProcessor`或`BasePostProcessor`，实现：
- `processor_name`: 处理器名称
- `_aprocess`: 异步处理方法

## 配置说明

### 主要配置项
| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| DEFAULT_TOP_K | int | 10 | 默认返回结果数量 |
| DEFAULT_TIMEOUT | int | 30 | 默认超时时间（秒） |
| FAIL_FAST | bool | False | 是否快速失败（部分组件失败时是否中断整个流程） |
| ENABLED_CHANNELS | list | ["vector_db", "mysql"] | 启用的渠道列表 |
| ENABLED_QUERY_PROCESSORS | list | ["context_processor", "query_rewriter", "intent_recognizer"] | 启用的查询处理器 |
| ENABLED_POST_PROCESSORS | list | ["result_filter", "deduplicator", "result_merger", "reranker"] | 启用的后处理器 |

### 渠道配置示例
```python
"vector_db": {
    "enabled": True,
    "timeout": 10,
    "collection": "product_knowledge",
    "weight": 1.0
}
```

## 常见问题

### Q: 如何添加新的检索渠道？
A: 继承`SearchChannel`接口实现自定义渠道，然后通过`search_engine.add_channel()`方法注册。

### Q: 如何调整结果排序规则？
A: 可以自定义重排序处理器，或修改`Reranker`的配置和权重规则。

### Q: 如何处理检索超时？
A: 可以通过配置`DEFAULT_TIMEOUT`或单个渠道的`timeout`参数调整超时时间，超时的渠道会被自动降级。
