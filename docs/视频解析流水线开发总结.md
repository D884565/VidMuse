# 视频解析流水线开发总结

## 一、项目概述
本项目实现了两套可扩展的视频内容解析流水线，基于插件化架构设计，能够灵活组合不同的处理单元完成复杂的内容分析任务。三套流水线分别针对视频片段理解、商品信息抽取、视频整体分析三个核心场景，为短视频内容的结构化解析提供了完整的解决方案。

## 二、架构设计
### 2.1 核心架构
采用三层插件化架构设计：
- **核心抽象层**：定义了统一的上下文(Context)、处理器(Processor)和流水线(Pipeline)抽象接口，所有组件都基于这些接口开发，保证了架构的一致性和扩展性。
- **处理器层**：实现了具体的业务处理逻辑，每个处理器只负责单一功能，可以自由组合到不同的流水线中。
- **流水线层**：将多个处理器按照业务逻辑顺序组装起来，形成完整的处理流程。

### 2.2 核心组件
#### 2.2.1 PipelineContext 上下文
- 用于在处理器之间传递数据和状态
- 支持业务数据存储、错误信息收集、元数据存储
- 提供统一的get/set接口，保证数据传递的一致性

#### 2.2.2 BaseProcessor 处理器抽象
- 所有处理器的基类，定义了统一的process接口
- 每个处理器只需要实现process方法，接收上下文并返回修改后的上下文
- 处理器之间完全解耦，可以独立开发、测试和复用

#### 2.2.3 BasePipeline 流水线抽象
- 负责组装和执行多个处理器
- 统一的错误处理机制，某个处理器出错时自动终止后续执行
- 标准化的结果输出格式，包含成功标记、数据、错误信息和元数据

## 三、两条条流水线功能说明
### 3.1 视频解析流水线（VideoParsingPipeline）- 端到端整合版
**流程**：视频拆分 → 分片理解 → 切片JSON生成 → 分片结构校验 → 结果聚合 → 整体理解 → 整体JSON生成 → 整体结构校验
- **输入**：视频文件路径、视频ID、视频时长
- **处理步骤**：
  1. `VideoSplitProcessor`：将长视频按固定时长拆分为多个短视频片段（Mock实现，可替换为真实的视频拆分逻辑）
  2. `VideoUnderstandingProcessor`：调用大模型接口分析每个视频片段的内容，输出结构化信息
  3. `SliceGenerateProcessor`：按照slice.json模板生成每个片段的JSON文件
  4. `SchemaValidationProcessor`：使用slice_valid.json的JSON Schema校验切片文件结构
  5. `VideoAggregationProcessor`：聚合所有校验通过的分片结果
  6. `VideoOverallUnderstandingProcessor`：基于分片信息对整个视频进行整体分析，生成视频级别的结构化信息
  7. `VideoGenerateProcessor`：生成video.json整体分析文件
  8. `SchemaValidationProcessor`：使用video_valid.json校验整体视频结构
- **输出**：
  - 每个视频片段对应一个slice.json文件，包含片段的结构化信息
  - 一个完整的video.json文件，包含视频的整体分析结果
- **特点**：完整的端到端处理流程，一次调用即可获得分片和整体的全部分析结果

### 3.2 商品解析流水线（ProductParsingPipeline）
**流程**：图文理解 → 商品信息生成 → 结构校验
- **输入**：图文混合的多模态内容、商品ID
- **处理步骤**：
  1. `ProductUnderstandingProcessor`：调用大模型分析图文内容，抽取商品的结构化信息
  2. `ProductGenerateProcessor`：生成product.json文件
  3. `SchemaValidationProcessor`：使用product_valid.json校验结构
- **输出**：商品信息product.json文件，包含商品的完整属性信息



## 四、使用示例
### 4.1 基本使用方式
```python
from backend.v1.app.rag.core.pipline import VideoParsingPipeline

# 创建流水线实例
pipeline = VideoParsingPipeline()

# 执行流水线
result = pipeline.run({
    "video_id": "v_001",
    "video_path": "/path/to/video.mp4",
    "video_duration": 30000  # 30秒视频
})

# 处理结果
if result["success"]:
    print("处理成功！")
    print(f"生成{len(result['data']['valid_slices'])}个有效切片")
else:
    print(f"处理失败：{result['errors']}")
```

### 4.2 自定义处理器
```python
from backend.v1.app.rag.core.pipline import VideoParsingPipeline
from backend.v1.app.rag.core.pipline.processors import CustomProcessor

# 使用自定义处理器替换默认处理器
pipeline = VideoParsingPipeline(custom_processors=[
    CustomProcessor1(),
    CustomProcessor2(),
    ...
])
```

## 五、代码优化点
### 5.1 路径处理优化
- 改进了项目根目录查找逻辑，通过识别.git、requirements.txt等标志性文件准确找到项目根目录
- 替代了原有的硬编码层数向上查找的方式，适配不同的项目部署结构
- 增加了回退机制，当无法找到标志性文件时回退到查找最近的resources目录

### 5.2 错误处理优化
- 所有字段访问使用get方法，避免KeyError
- 优化了校验失败时的错误信息处理，当slice_id不存在时使用"unknown"作为默认值
- 统一的异常处理机制，保证流水线执行的稳定性

### 5.3 通用性优化
- SchemaValidationProcessor支持任意JSON Schema的校验，通过传入不同的schema_path可以处理不同类型的文件校验
- 处理器完全解耦，可以在不同流水线之间自由复用
- 动态路径适配，支持Windows、Linux等多操作系统
